# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 6.x     | :white_check_mark: |
| < 6.0   | :x:                |

## Reporting a Vulnerability

**Please do NOT report security vulnerabilities through public GitHub issues.**

Instead, please report them by email to **info@hexaglue.io**.

Include as much of the following information as possible:

- Type of issue (e.g., dependency vulnerability, code injection, information disclosure)
- Full paths of source file(s) related to the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue

## Response Timeline

- **48 hours**: Acknowledgment of your report
- **7 days**: Initial assessment and severity determination
- **14 days**: Fix timeline communicated

## Coordinated Disclosure

We follow coordinated disclosure. We ask that you:

1. Allow us reasonable time to investigate and address the issue before public disclosure
2. Make a good faith effort to avoid privacy violations, data destruction, and service disruption
3. Do not access or modify other users' data

## Scope

This policy applies to the HexaGlue code generation engine and its official plugins:

- `hexaglue-core`
- `hexaglue-arch`
- `hexaglue-syntax`
- `hexaglue-spi`
- `hexaglue-maven-plugin`
- `hexaglue-plugin-jpa`
- `hexaglue-plugin-rest`
- `hexaglue-plugin-living-doc`
- `hexaglue-plugin-audit`

Third-party integrations and generated code are outside the scope of this policy.
